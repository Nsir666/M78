一、FreeRTOS 任务划分

任务名称优先级栈大小运行周期功能描述操作的外设
Task_Sensor4 (最高)512 Bytes200 ms采集所有传感器数据，写入共享结构体DS1302 / DS18B20 / ADXL345 / MAX30102
Task_Display3512 Bytes100 ms从共享结构体读取数据，刷新OLED双页面显示OLED SSD1306 (I2C1)
Task_Alarm3256 Bytes100 ms监测心率/血氧/体温阈值，处理闹钟/定时/里程提醒BEEP 蜂鸣器 (PB9)
Task_Key2256 Bytes20 ms扫描5个按键，消抖，发送按键事件到队列KEY0~4 (PB12~15, PA8)
Task_Bluetooth1 (最低)256 Bytes事件驱动接收蓝牙串口命令，上报传感器数据USART1 (PA9/PA10, 9600bps)

二、任务优先级设计

优先级任务设计理由
4 (最高)Task_Sensor数据采集不能被延迟，否则传感器FIFO溢出
3Task_Display + Task_Alarm同级时间片轮转，显示与报警互不阻塞
2Task_Key按键需及时响应，但不影响传感器采集
1 (最低)Task_Bluetooth通信耗时大，后台处理不阻塞实时功能

三、任务间通信与资源保护

机制名称数量用途数据流向
互斥锁g_dataMutex1保护 g_data 共享数据结构读写Task_Sensor → 写 / 其他任务 → 读
互斥锁g_oledMutex1保护 OLED 显示，避免多任务同时写屏所有需要写OLED的任务
队列g_keyQueue1 (深度8)按键事件从按键任务传递到报警任务Task_Key → Task_Alarm
二值信号量g_uartRxSem1UART IDLE中断通知蓝牙任务处理数据USART1_IRQ → Task_Bluetooth
